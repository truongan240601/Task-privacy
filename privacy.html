Privacy Policy for LTA Chat to I-TASK / i-request

Last updated: May 25, 2026

Overview

LTA Chat to I-TASK / i-request is a Chrome extension built for authorized internal LTA users. Its single purpose is to help users turn selected Element chat text into an I-TASK PRO task or prefill an i-office i-request form for user review.

Data the extension processes

The extension may process the following data only when needed for its core workflow:

  • Selected chat text from the active Element chat page after the user starts an extension action.
  • Page context such as the chat title or current internal application page.
  • Internal project, department, issue, task, and request metadata returned by LTA systems.
  • Local workflow status, cached master data, recent selections, and non-personal UX metrics stored in Chrome local storage.
  • Best-effort audit information needed to record task or request creation attempts in LTA internal systems.

How the data is used

The extension uses data only to provide the user-facing feature described above:

  • Create an I-TASK PRO task from selected chat text after user confirmation.
  • Prefill an i-office i-request form from selected chat text for user review.
  • Load internal project, department, and issue options from authenticated LTA sessions.
  • Show confirmation dialogs, progress messages, local task status, and recent workflow results.
  • Write internal audit records for task/request creation attempts.

Data storage

The extension stores limited operational data in chrome.storage.local, such as cached project/request options, recent selections, latest workflow status, and local non-personal UX metrics. Selected chat text is used for the requested workflow and is not used for advertising or sale.

Data sharing

The extension sends data only to authorized LTA internal systems required for the workflow, including I-TASK PRO, i-office, and LTA audit APIs. The extension does not sell user data, does not use user data for advertising, and does not transfer user data to third parties for unrelated purposes.

Remote code

The extension does not execute remote JavaScript or WebAssembly. Extension code is packaged with the Chrome extension.

User control

The extension runs its main workflows only after an explicit user action, such as selecting chat text and choosing an extension context-menu command. The i-office i-request workflow only pre-fills the request form; it does not submit the request automatically.

Security

The extension is restricted to declared LTA internal domains. It relies on the user's existing authenticated browser sessions for LTA systems and does not embed production credentials in the extension package.

Contact

For privacy or support questions about this extension, contact the extension publisher through the Chrome Web Store developer account or the responsible LTA internal support channel.
